# Harvest brief — loeliger-til (INJECT VERBATIM)
WHAT COUNTS
- keyword_entry: a centered ALLCAPS word-name followed by labeled fields (Class/Function/Input-Output/Usage/Code/Bytes[/Formal Definition]). One entity per entry.
- concept: a named TIL mechanism taught in prose (threading, inner interpreter, vocabularies, defining words, virtual memory).
- figure: a "Figure N.M: <caption>" line-art diagram. Describe only, never reproduce.
- listing: a "Listing N.M: <caption>" code block.
NAMING WEATHER
- Word-names carry sigils that ARE part of the name: $ (machine-code subroutine), * (compiler literal), : ; @ ! # < > + . Keep sigils; never strip.
- Aliases: a keyword may be discussed in prose by concept-name and defined in ch6 by symbol-name; treat as same-entity candidates but do not merge (arbitration owns it).
NOISE THAT IS CONTENT / CONTENT THAT LOOKS LIKE NOISE
- The block glyph in Formal Definition lines is the TIL token SEPARATOR (a space). OCR renders it as "B" or leaves it as a block; it is NOT hex B and NOT a letter. Read ":<sep>APART<sep>SPACE<sep>8..." as tokens.
- Z80 mnemonics in Code blocks (POP PUSH LD INC AND OR GET) are assembly, NOT TIL keywords. Do not mint entities from them.
- (HL) may appear as {HL}; a comment ";" may appear as a bullet/asterisk.
SKIP ON SIGHT
- running headers (chapter title + page number at page top)
- the back-matter Index and Bibliography (out of harvest scope)
LOCAL CITATION FORM
- page N  and section §N.M[.K]; figures fig:N-M; listings listing:N-M.
