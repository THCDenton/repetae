## Ground truth learned this session

- **W10: a deliverables note is not a receipt.** A file listed at a close
  can silently fail to land; the NEXT boot's manifest check is the actual
  receipt — and it worked, catching the miss one session later at zero
  drama. Third distinct filing/mount defect caught by the schema-v2.1 loop
  in three booted sessions (two W9 latency events, one W10 missed filing).
  The correction was cheap **because the patch spec was durable** — record
  patch specs, not just patch facts.
- **The interruption class is survivable.** A platform "session limit" hit
  mid-close; on resume the container had persisted (draft intact, verified
  by ls/wc/header before claiming continuity). Handling: verify state,
  never assume it — the same discipline as the boot loop, applied
  mid-session.
- **The docket → explanation → wholesale-ruling pattern scales down.** It
  worked for a 12-item charter amendment (07-12c) and for an 8-item
  deviations read-back (today). Adopt as the default form for any
  multi-item ratification.
