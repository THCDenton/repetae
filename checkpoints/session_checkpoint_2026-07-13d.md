## Ground truth learned this session

- **Optimizing identity for the model exported toil to the operator.** The
  07-13b scheme made version identity clean on Claude's side and created a
  mandatory delete-on-every-refile burden plus the entire W9 impersonation
  class on the operator's side. Process law must be costed on BOTH sides
  of the interface; the operator's filing workflow is part of the system.
- **Inert beats absent.** The durable fix was not better deletion
  discipline but making stale copies harmless — a file whose name says
  what it is cannot impersonate the current one. Same shape as the
  honest-TBD posture: prefer states that are safe-by-construction over
  procedures that must be executed perfectly.
- **A reversed ruling needs its reversal recorded where the original
  lives.** The 07-13b rejection rationale ("filename bumps create
  retirement debt") was written into the record; the reversal is now
  written next to it (lexicon §10, primer v4, here) with the reframe that
  dissolved it. Future sessions can see WHY, not just WHAT.
- **Scaffolding is the model-handoff strategy.** Boot manifests, mechanical
  rev derivations, verified-how ledgers, self-tests, and the docket +
  arguments pass all catch model error mechanically — a handoff to a
  different model degrades gracefully inside them. The one addition the
  handoff needed was observability: log the model, treat the first boot as
  a probe, simplify on stumble.
- **Rev-counter derivations verified mechanically at both closes** (12
  changelog bullets = rev 12; 7 banner stamps + 1 = rev 8) — the counter's
  own rule, applied as a required check, third session running it has
  either passed clean or caught a defect. Keep it a close-step forever.
