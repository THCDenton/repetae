Wire schemas, hop by hop

The data shape at every arrow. Grammars marked PRIOR ART are BJJ-era and their generalized successors may amend them — recorded here so the diagram never silently upgrades an ancestor into law.

H1 — operator → Stage 0

Raw source as a direct chat attachment (zip if transformed). Never via project knowledge — the platform silently replaces PDFs with lossy text.

H2 — registration line (sources.md)

- <slug> | <type> | <title / author / year> | loc-grammar: <grammar>

Sample: - sailing-for-dummies | book | Sailing For Dummies, 2nd Edition / J.J. Isler & Peter Isler / 2006 | loc-grammar: page:N (parent chapter-N)

H3 — ingest → substrate → fidelity gate

Plain-text substrate + measurements (lines / words / book pages). Gate verdict is per content class, ruled by the operator (sailing precedent: PASS prose, FAIL tabular + figure; accepted as-is under the split verdict).

H4 — discovery ↔ operator

Button decisions with the re-ask rule; every convention line lands with one of the five provenance tags; unanswerable questions leave as escalations/ files, never silent defaults.

H5 — discovery output family → consumers

No artifact is consumed by everyone: master doc → humans + sessions (almost never workers) · harvest brief (≤25 lines) → worker packets verbatim · content-mode map (mode ∈ prose | tabular | dialogic | code_listing | mixed, no improvised values) → packet builder · loc anchors → merge + citation resolution · registry queue → registry_dossier · kit nominations → workshop (kit assembly) · config fragment → schema/config sessions ONLY (never a live-config shortcut; the v2-proposed pile was fully triaged into schema v2 on 07-13b — future discovery proposals are fenced as v3-proposed) · arbitration seed (v2.5) → the harvest arbitration layer's boot state only (judge-visible under DAG, integrator-visible under fold; tiebreak rung 2; NEVER the brief, NEVER a worker packet) · chunk plan CSV (v2.5) → packet builder + driver (defines units; any overlap policy is topology-owned and consumes them).

H6 — substrate → spans

Slicing owned by the driver. span_size / span_overlap are RETIRED as constants (v1.2 item 8; schema disposition executed 07-13b) — the per-source chunk block (schema v2 §3.2.1: ruled boundary, measured fallback bound, overrides, plan CSV) replaces them; kit spans may still use provisional sizes stamped fixture-not-policy. Span-sizing ruling (RATIFIED 07-11b): parameterize — size stays a config dial and the worker carries boundary doctrine instead of a baked-in size. Overlap floor (correctness precondition, [model-knowledge, unverified]): overlap must be at least the largest single-entity footprint the source shows, or some entity is seen whole nowhere and no downstream step can recover it. Overlap is a correctness floor, not only an efficiency dial.
